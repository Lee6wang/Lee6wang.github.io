## Introduction

The transfer function is usually the first tool encountered when learning control theory. For a linear time-invariant system, under zero initial conditions, it is written as

$$
G(s)=\frac{Y(s)}{U(s)}
$$

Then stability and dynamic performance can be analyzed through poles, zeros, root locus, and frequency response. For low-order single-input single-output systems, this method is intuitive, efficient, and very practical in engineering.

But when the controlled object becomes a real motor system, a natural question appears: **can transfer functions alone still describe the essence of a motor control system completely?**

A motor is not just an input-output black box. It is a dynamic system with coupled electrical and mechanical parts. Inside it, current, speed, position, torque, disturbance, and other important variables evolve together. The focus is no longer only "how the input affects the output", but "how the internal state of the system evolves".

---

## 1 Why Transfer Functions Are Useful in Motor Control

### 1.1 Local Subsystems Naturally Fit Transfer Functions

At the introductory stage of motor control, transfer functions are indeed enough in many places. The reason is simple: under suitable assumptions, many local subsystems can be approximated as low-order linear objects.

**Current loop:** In the dq coordinate frame, after ignoring cross coupling and back-EMF compensation error, the current dynamics can be approximated as

$$
L\frac{di}{dt}+Ri=u \quad\Longrightarrow\quad G_i(s)=\frac{1}{Ls+R}
$$

This is a first-order inertial object, so PI control and bandwidth tuning can be applied directly.

**Speed loop:** The mechanical side satisfies

$$
J\frac{d\omega}{dt}=T_e-T_L-B\omega \quad\Longrightarrow\quad G_\omega(s)=\frac{1}{Js+B}
$$

This is also a first-order inertial object and can be designed with classical methods.

**Layered dual closed loop:** If the current-loop bandwidth is much higher than the speed-loop bandwidth, then $i_q \approx i_q^*$, and therefore $T_e \approx K_t i_q^*$. The speed loop can treat the current loop as an ideal actuator and continue to use a single-input single-output design method for the outer loop. This time-scale separation, "fast inner loop and slow outer loop", is exactly the basis for the classical dual closed-loop structure.

### 1.2 Advantages of Transfer Functions

Transfer functions are friendly to beginners and engineers alike. They directly connect control effect with dynamic response, such as rise time, overshoot, and bandwidth. They turn differential equations into algebraic problems, and they correspond directly to a large amount of classical engineering experience.

Therefore, transfer functions are not outdated. In local current-loop and speed-loop design, they are still the most direct and efficient analysis tool today.

## 2 Why Motor Systems Eventually Need Another Tool

### 2.1 A Motor Is Not a Single-Variable Problem

Take a PMSM as an example. The system contains dynamic variables such as $i_d$, $i_q$, $\omega$, and $\theta$ at the same time, and these variables have clear coupling and hierarchy: current determines electromagnetic torque, torque determines acceleration, speed affects position, position participates in coordinate transformation, and load disturbance enters the mechanical equation directly. The behavior of the system at each instant is jointly determined by a set of states, not by a single variable.

### 2.2 Several Reasons Transfer Functions Start to Become Insufficient

**Multivariable coupling is hard to express.** The dq voltage equations of a PMSM are:

$$
L_d\frac{di_d}{dt}=u_d-Ri_d+\omega_e L_q i_q
$$

$$
L_q\frac{di_q}{dt}=u_q-Ri_q-\omega_e L_d i_d-\omega_e\psi_f
$$

The d-axis and q-axis are naturally coupled. If a single transfer function is used for analysis, many simplifications are required and structural information about the system is lost.

**The linearization assumptions are limited.** A transfer function requires the system to be approximately linear, with unchanged parameters, a fixed operating point, and zero initial conditions. A real motor has changing operating conditions, parameter drift, nonlinear coordinate transformation, back EMF, and coupling terms. The transfer function only holds near a local operating point.

**Initial states are discarded.** Initial current, speed, and position all influence subsequent dynamics. However, $G(s)=Y(s)/U(s)$ is inherently a mapping under zero initial conditions, so the effect of initial states is weakened.

**The viewpoint is limited.** A transfer function studies the channel $u \rightarrow y$, but the real problem faced by a motor controller is: under inputs and disturbances, **how will the internal state of the system evolve**, and can these states converge to the desired trajectory? This is no longer a question that one channel can answer.

## 3 Modern Control Basics: What State Space Really Is

Since we are moving toward state space, it is useful to organize the basic concepts first.

### 3.1 Core Idea: From "Channel" to "Evolution"

Classical control focuses on the mapping from input to output. Given $u(s)$, it passes through $G(s)$ and produces $y(s)$. Modern control focuses on how the internal state of the system evolves over time: driven by the input, how does a group of internal variables change step by step?

A **state** is the minimum set of information needed at a certain moment to completely determine the future behavior of the system. If the current state $x(t_0)$ and the future input $u(t)$ for $t \geq t_0$ are known, then the entire future trajectory of the system can be uniquely determined.

As an analogy, a transfer function is like looking at a car's throttle-speed curve. State space is like watching the engine speed, gearbox gear, tire grip, and vehicle attitude at the same time. You not only know how the speed will change, but also **why** it changes that way.

### 3.2 State Equation and Output Equation

A general nonlinear system can be written as two equations:

$$
\dot{x}=f(x,u,t) \qquad \text{(state equation: describes how the state changes over time)}
$$

$$
y=g(x,u,t) \qquad \text{(output equation: describes what can be observed)}
$$

Here $x \in \mathbb{R}^n$ is the state vector, $u \in \mathbb{R}^m$ is the input vector, and $y \in \mathbb{R}^p$ is the output vector.

If the system can be linearized near an operating point, the most common **linear time-invariant (LTI) state-space** form is obtained:

$$
\dot{x}=Ax+Bu
$$

$$
y=Cx+Du
$$

The four matrices each have a clear meaning:

- $A \in \mathbb{R}^{n \times n}$: the **system matrix**. It describes how states influence one another when there is no external input. Diagonal elements of $A$ reflect the decay or growth trend of each state itself, while off-diagonal elements reflect coupling between states. The structure of $A$ is basically the "skeleton" of the system dynamics.
- $B \in \mathbb{R}^{n \times m}$: the **input matrix**. It describes where the control input enters the system and which states it affects. Each column of $B$ corresponds to one input channel, and nonzero elements in that column indicate which states the input can directly act on.
- $C \in \mathbb{R}^{p \times n}$: the **output matrix**. It describes which linear combinations of states can be measured. If a state is measured directly, the corresponding row of $C$ is a unit vector.
- $D \in \mathbb{R}^{p \times m}$: the **feedthrough matrix**. It indicates whether the input affects the output directly without passing through state dynamics. In most physical systems, $D=0$.

### 3.3 Time-Domain Solution of a Linear System: State Transition

The LTI system $\dot{x}=Ax+Bu$ has the analytical solution:

$$
x(t)=e^{A(t-t_0)}x(t_0)+\int_{t_0}^{t}e^{A(t-\tau)}Bu(\tau)\,d\tau
$$

This formula is worth reading carefully because it clearly shows a key difference between state space and transfer functions:

- The first term $e^{A(t-t_0)}x(t_0)$ is the **free response**, completely determined by the initial state and independent of the input.
- The second term is the **forced response**, driven by the input $u(\tau)$.

A transfer function can only describe the second term, namely the input-output relation under zero initial conditions. State space naturally includes both parts. This is not a small detail in motor control: the current initial value at power-on and the speed initial value during startup both truly affect the subsequent dynamics.

The matrix exponential $e^{At}$ is called the **state transition matrix**, denoted as $\Phi(t)$. Its physical meaning is: if there is no input, after starting from the initial state $x(0)$, the system will reach $\Phi(t)x(0)$ after $t$ seconds. The behavior of $\Phi(t)$ is completely determined by the eigenvalues of $A$: eigenvalues with negative real parts correspond to decaying modes, while those with positive real parts correspond to divergent modes.

### 3.4 Mathematical Relationship with the Transfer Function

Apply the Laplace transform to an LTI system, using zero initial conditions:

$$
sX(s)-x(0)=AX(s)+BU(s)
$$

Let $x(0)=0$. Then $X(s)=(sI-A)^{-1}BU(s)$. Substitute this into the output equation $Y(s)=CX(s)+DU(s)$:

$$
G(s)=\frac{Y(s)}{U(s)}=C(sI-A)^{-1}B+D
$$

This formula reveals several important facts:

1. A transfer function can be derived from state space. It is a "projection" of the state-space model under zero initial conditions.
2. The reverse is not unique. Different combinations of $(A,B,C,D)$ can produce the same $G(s)$, so the internal structure cannot be uniquely reconstructed from the transfer function.
3. The poles of $G(s)$ are the roots of $\det(sI-A)=0$, namely the eigenvalues of $A$. But if some modes are uncontrollable or unobservable, they may be canceled in $G(s)$, forming so-called **hidden modes**. The transfer function may make the system look stable, while an unstable internal mode is actually hidden. This is a real trap of transfer-function analysis.

### 3.5 Stability Analysis

**LTI systems:** Stability is determined by the eigenvalues of the $A$ matrix, also called the system poles. If all eigenvalues of $A$ have negative real parts, the system is asymptotically stable, and any initial disturbance decays to zero over time.

Specifically, for eigenvalues $\lambda_i = \sigma_i + j\omega_i$ of $A$:

- $\sigma_i < 0$: a decaying mode; larger $|\sigma_i|$ means faster decay.
- $\sigma_i > 0$: a divergent mode; the system is unstable.
- $\omega_i \neq 0$: an oscillatory mode; the frequency is determined by $\omega_i$.

This is essentially the same as pole analysis in transfer functions. The advantage of state space is that it can see all modes, including hidden modes that may be canceled in the transfer function.

**Nonlinear systems:** Transfer functions are basically powerless for nonlinear systems because the superposition principle no longer holds. State space can use **Lyapunov stability theory** instead. One constructs a scalar function $V(x)$, called a Lyapunov function. If it is positive definite and decreases along system trajectories, meaning $\dot{V}<0$, system stability can be proven without solving the differential equations.

The power of the Lyapunov method is that it works entirely in state space. It does not require linearization or a closed-form solution. For the nonlinearities and parameter uncertainties that appear frequently in motor control, it is an extremely important tool.

### 3.6 Controllability: Can the Input "Reach" All States?

**Controllability** answers this question: through a suitable input $u$, can the system be driven from any initial state to any target state within finite time?

For an LTI system, the criterion is that the **controllability matrix**

$$
\mathcal{C}=\begin{bmatrix} B & AB & A^2B & \cdots & A^{n-1}B \end{bmatrix}
$$

has full rank, that is $\text{rank}(\mathcal{C})=n$.

Intuitively, the column space of $B$ represents the state directions that the input can directly influence in one step. $AB$ represents the directions that the input can indirectly reach after one step of state evolution. $A^2B$ is after another step, and so on. If all these directions together span the whole $n$-dimensional space, then the input can eventually cover all state dimensions.

In motor control, controllability means: through the two voltage inputs $u_d$ and $u_q$, can $i_d$, $i_q$, $\omega$, and $\theta$ be adjusted independently? If a state is uncontrollable, no controller can arbitrarily regulate it.

### 3.7 Observability: Can All States Be "Seen" from the Output?

**Observability** answers this question: through the observed output $y$, can the initial state of the system be uniquely determined within finite time?

The criterion is that the **observability matrix**

$$
\mathcal{O}=\begin{bmatrix} C \\ CA \\ CA^2 \\ \vdots \\ CA^{n-1} \end{bmatrix}
$$

has full rank, that is $\text{rank}(\mathcal{O})=n$.

Intuitively, $C$ represents the state components that can be measured directly. $CA$ represents the new state information contained in the measurement after one step of evolution, and so on. If the collected information is enough to distinguish any two different initial states, the system is completely observable.

In motor control, this concept is very practical. The core problem of sensorless control is: using only voltage and current signals, can rotor position $\theta$ and speed $\omega$ be reconstructed? If the system is unobservable under some operating condition, such as zero speed or very low speed where the back EMF is close to zero, the observer loses its estimation capability. This is not merely an algorithm issue; it is a limitation of the system structure itself.

### 3.8 State Feedback and Pole Placement

One of the most direct control applications of the state-space framework is **state feedback**. Suppose all states can be measured, or estimated by an observer, and let

$$
u = -Kx + r
$$

where $K \in \mathbb{R}^{m \times n}$ is the feedback gain matrix and $r$ is the reference input. Substituting it into the state equation gives

$$
\dot{x} = (A-BK)x + Br
$$

The closed-loop system matrix becomes $A_{cl}=A-BK$. If the system is controllable, $K$ can be chosen to place the eigenvalues of $A_{cl}$, namely the closed-loop poles, at any desired locations in the complex plane.

This is the basic idea of **pole placement**: through state feedback, the dynamic characteristics of the closed-loop system, including decay speed and oscillation frequency, can be specified by design.

Compared with classical PI/PID, pole placement has the advantage that it acts directly on all states, not only on the error of one input-output channel. For multivariable coupled systems, this design method is more systematic.

### 3.9 Observer: When States Cannot Be Measured Directly

In real systems, not every state has a sensor. For example, in motor control, current is usually measurable, but rotor position and speed may need to be estimated in sensorless schemes. Load disturbance is even completely unknown.

The idea of the **Luenberger observer** is to build a "shadow system" with the system model to simulate the real system, and then use the difference between the actual output and the model output to correct the estimate:

$$
\dot{\hat{x}} = A\hat{x} + Bu + L(y - C\hat{x})
$$

Here $\hat{x}$ is the state estimate, and $L \in \mathbb{R}^{n \times p}$ is the observer gain. Define the estimation error $e = x - \hat{x}$, then

$$
\dot{e} = (A - LC)e
$$

If the system is observable, $L$ can be chosen so that all eigenvalues of $A-LC$ lie in the left half-plane, making the estimation error converge to zero at a desired speed.

Observer design and state feedback are actually dual problems. State feedback adjusts the eigenvalues of $A-BK$ through $K$ and requires controllability. The observer adjusts the eigenvalues of $A-LC$ through $L$ and requires observability.

In motor control, observers are widely used. Extended Kalman filters (EKF) and sliding-mode observers in sensorless FOC are essentially using system observability to reconstruct unmeasured states.

### 3.10 What State Space Provides

Summarizing the discussion above, state space provides a complete analysis and design framework beyond transfer functions:

|            | Transfer Function | State Space |
| ---------- | ----------------- | ----------- |
| Object described | One input-output channel | Internal dynamics of the whole system |
| Initial conditions | Must be zero | Naturally included |
| Multivariable systems | Requires transfer matrices, structure is not intuitive | Naturally supported |
| Nonlinear systems | Basically cannot handle them | Can describe them directly |
| Stability | Pole analysis, possibly missing hidden modes | Eigenvalue analysis plus Lyapunov theory |
| Controllability/observability | Cannot judge directly | Has systematic criteria |
| Control design | PI/PID and frequency-domain shaping | State feedback, pole placement, optimal control, observers |

## 4 PMSM Example: A Complete Derivation from Physical Equations to State Space

The previous sections covered general theory. Now return to the motor system and walk through the complete process from physical equations to a state-space matrix.

### 4.1 Write the Physical Equations

In the synchronous rotating dq coordinate frame, the voltage equations of a PMSM are:

$$
u_d = Ri_d + L_d\frac{di_d}{dt} - \omega_e L_q i_q
$$

$$
u_q = Ri_q + L_q\frac{di_q}{dt} + \omega_e L_d i_d + \omega_e \psi_f
$$

The mechanical equation is:

$$
J\frac{d\omega}{dt} = T_e - T_L - B\omega
$$

where the electromagnetic torque is $T_e = \frac{3}{2}p[\psi_f i_q + (L_d - L_q)i_d i_q]$. For a nonsalient motor where $L_d = L_q$, it simplifies to $T_e = \frac{3}{2}p\psi_f i_q$.

The kinematic equation is:

$$
\frac{d\theta}{dt} = \omega
$$

The relation between electrical angular speed and mechanical angular speed is $\omega_e = p\omega$, where $p$ is the number of pole pairs.

### 4.2 Rearrange into Standard State-Equation Form

Rearrange the equations above into the form "$\dot{x} = \ldots$":

$$
\frac{di_d}{dt} = -\frac{R}{L_d}i_d + \frac{\omega_e L_q}{L_d}i_q + \frac{1}{L_d}u_d
$$

$$
\frac{di_q}{dt} = -\frac{\omega_e L_d}{L_q}i_d - \frac{R}{L_q}i_q - \frac{\omega_e \psi_f}{L_q} + \frac{1}{L_q}u_q
$$

$$
\frac{d\omega}{dt} = \frac{K_t}{J}i_q - \frac{B}{J}\omega - \frac{T_L}{J}
$$

$$
\frac{d\theta}{dt} = \omega
$$

where $K_t = \frac{3}{2}p\psi_f$ for a nonsalient motor.

Define the state vector, input vector, and disturbance:

$$
x = \begin{bmatrix} i_d \\ i_q \\ \omega \\ \theta \end{bmatrix}, \quad
u = \begin{bmatrix} u_d \\ u_q \end{bmatrix}, \quad
d = T_L
$$

This is already a complete nonlinear state equation $\dot{x} = f(x, u, d)$, because $\omega_e = p\omega$ is itself a function of the state variable $\omega$, which causes products between states such as $\omega \cdot i_d$ and $\omega \cdot i_q$ to appear in the equations.

### 4.3 Linearize Near an Operating Point

To obtain an LTI form, choose a steady operating point $(i_{d0}, i_{q0}, \omega_0, \theta_0)$, define deviation variables such as $\Delta i_d = i_d - i_{d0}$, and apply a first-order Taylor expansion to the nonlinear equations.

For a common operating point under the $i_d = 0$ control strategy, $i_{d0}=0,\, i_{q0}=i_{q0},\, \omega_0=\omega_0$. After linearization:

$$
\frac{d}{dt}\begin{bmatrix} \Delta i_d \\ \Delta i_q \\ \Delta\omega \\ \Delta\theta \end{bmatrix}=
\underbrace{\begin{bmatrix}
-\frac{R}{L_d} & \frac{p\omega_0 L_q}{L_d} & \frac{p L_q i_{q0}}{L_d} & 0 \\
-\frac{p\omega_0 L_d}{L_q} & -\frac{R}{L_q} & -\frac{p(L_d i_{d0}+\psi_f)}{L_q} & 0 \\
0 & \frac{K_t}{J} & -\frac{B}{J} & 0 \\
0 & 0 & 1 & 0
\end{bmatrix}}_{A}
\begin{bmatrix} \Delta i_d \\ \Delta i_q \\ \Delta\omega \\ \Delta\theta \end{bmatrix}
+
\underbrace{\begin{bmatrix}
\frac{1}{L_d} & 0 \\
0 & \frac{1}{L_q} \\
0 & 0 \\
0 & 0
\end{bmatrix}}_{B}
\begin{bmatrix} \Delta u_d \\ \Delta u_q \end{bmatrix}
$$

Substituting $i_{d0}=0$ can further simplify the $A$ matrix.

This is the linear state-space model of a PMSM near a certain operating point. In the $A$ matrix, the physical meaning is clear: diagonal elements are self-decay terms for each state, such as $-R/L$ and $-B/J$; off-diagonal elements are coupling terms. The d-axis and q-axis are coupled through the speed $\omega_0$, and the q-axis current drives the mechanical dynamics through $K_t/J$.

### 4.4 Read Physical Information from the Matrices

After obtaining the $A$ and $B$ matrices, a lot of physical information can be read directly.

**Input channels:** The $B$ matrix shows that $u_d$ directly acts only on $i_d$, and $u_q$ directly acts only on $i_q$. Voltage inputs do not act directly on mechanical states; they must influence $\omega$ and $\theta$ indirectly through electromagnetic torque.

**Coupling structure:** The $(1,2)$ and $(2,1)$ entries of the $A$ matrix are cross-coupling terms whose magnitudes are proportional to $\omega_0$. This explains why dq-axis coupling becomes more severe at high speed and why decoupling compensation becomes more important.

**Drive chain:** Starting from $B$, the input first reaches the current states, which are the first two rows of $B$. Current then drives speed through the $(3,2)$ element $K_t/J$ in $A$, and speed drives position through the $(4,3)$ element 1 in $A$. This is the cascade structure of "current -> torque -> speed -> position", visible directly in the matrix.

**Disturbance entry:** The load disturbance $T_L$ enters the $\dot{\omega}$ equation with coefficient $-1/J$. If the disturbance is also written in matrix form, it affects only the third state. This means the disturbance enters from the mechanical side and affects the electrical side indirectly through the cascade relationship.

### 4.5 Simplified Case: Current Subsystem

In many analyses, if only the current-loop design is considered, $\omega$ can be regarded as a slowly varying parameter because the mechanical time constant is much larger than the electrical time constant. The current subsystem is:

$$
\frac{d}{dt}\begin{bmatrix} i_d \\ i_q \end{bmatrix}=
\begin{bmatrix}
-R/L_d & \omega_e L_q/L_d \\
-\omega_e L_d/L_q & -R/L_q
\end{bmatrix}
\begin{bmatrix} i_d \\ i_q \end{bmatrix}
+
\begin{bmatrix}
1/L_d & 0 \\
0 & 1/L_q
\end{bmatrix}
\begin{bmatrix} u_d \\ u_q \end{bmatrix}
+
\begin{bmatrix} 0 \\ -\omega_e\psi_f/L_q \end{bmatrix}
$$

This is a two-input, two-state linear system, with $\omega_e$ treated as a parameter. The off-diagonal elements of the $A$ matrix are exactly the cross-coupling terms that require feedforward compensation, and the final term is the back-EMF disturbance. Decoupling compensation in FOC is essentially eliminating the off-diagonal elements of the $A$ matrix and the back-EMF term, so that the d-axis and q-axis become two independent first-order systems. At that point, returning to the transfer-function method is completely reasonable.

This also shows that transfer functions and state space are not contradictory: **state space helps you see the global structure clearly, while transfer functions help you design efficiently on simplified decoupled channels.**

### 4.6 The dq Transformation Itself Is State-Variable Thinking

The transformation from abc coordinates to dq coordinates is essentially converting AC phase variables into state variables more suitable for control analysis. FOC already contains state-space thinking implicitly: first choose key state variables $i_d$ and $i_q$, then design decoupling and control laws around those variables.

### 4.7 The Key Problems in Motor Control All Point to States

Decoupling control compensates the off-diagonal elements of the $A$ matrix. The current inner loop regulates the state dynamics of $i_d$ and $i_q$. The speed outer loop regulates the state dynamics of $\omega$. Observer design uses observability to estimate unmeasured states such as position, speed, and disturbance. More advanced robust and nonlinear control methods are built directly on state space.

For PMSM control, state space is not an additional tool; it is a more essential way to understand the control structure.

---

## 5 Summary

Moving from transfer functions to state space does not mean transfer functions fail. It means the object of analysis has changed: **from channel to system**.

For linear, low-order, single-loop problems, transfer functions are still the best choice. They are compact, and poles, zeros, and bandwidth are clear at a glance. PI design and frequency-domain tuning still rely on them. But when the system contains multiple coupled states, multiple inputs and outputs, nonlinear operating conditions, observers, or state feedback, the advantages of state space become irreplaceable.

A more reasonable understanding is: **transfer functions** are suitable for analyzing a specific control channel, while **state space** is suitable for analyzing the whole dynamic system. Simple problems use the channel viewpoint; complex motor systems use the system viewpoint. They are not opposing methods, but description tools at different levels.

Once a system is written in state-space form, the control problem can be further transformed into a problem of error dynamics and state convergence. This also opens the door to later exploration of more advanced control methods.
